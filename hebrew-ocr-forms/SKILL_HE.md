### שלב 3: הרצת OCR בעברית עם Tesseract

תסתכלו על `scripts/extract_form_fields.py` לפייפליין החילוץ המלא.

**הגדרות Tesseract לטפסים בעברית:**
```python
config = (
    '--oem 1 '          # LSTM neural net (best for Hebrew)
    '--psm 6 '          # Assume uniform block of text
    '-l heb+eng '       # Hebrew + English (forms have both)
    '-c preserve_interword_spaces=1'  # Keep spacing for field alignment
)
```

- לטפסים טבלאיים (טאבו, טופס 106) תשתמשו ב-PSM 4 במקום PSM 6
- תמיד במצב LSTM (--oem 1) לדיוק הטוב ביותר בעברית
- תכללו גם heb וגם eng כי טפסים מכילים גם עברית וגם אנגלית/מספרים

### שלב 5: טיפול ב-RTL וטקסט דו-כיווני

```python
import unicodedata

def normalize_bidi_text(text):
    """Normalize bidirectional text from Hebrew OCR output."""
    lines = text.split('\n')
    normalized = []
    for line in lines:
        # Strip bidi control characters
        clean = ''.join(
            c for c in line
            if unicodedata.category(c) != 'Cf'
        )
        clean = ' '.join(clean.split())
        if clean:
            normalized.append(clean)
    return '\n'.join(normalized)
```

## משאבים מצורפים

### סקריפטים
- `scripts/preprocess_image.py` - הכנת תמונות של טפסים ישראליים סרוקים ל-OCR: המרה לגווני אפור, יישור סריקות מסובבות, בינאריזציה מסתגלת לתאורה לא אחידה, הסרת רעש מורפולוגית, שיפור ניגודיות CLAHE אופציונלי, והסרת שוליים. הרצה: `python scripts/preprocess_image.py --help`
- `scripts/extract_form_fields.py` - הרצת Tesseract OCR בעברית על תמונות טפסים מעובדות וחילוץ שדות מובנים לפי סוג הטופס. תומך בזיהוי אוטומטי של טאבו, טופס 106 וטפסי ממשלה ישראליים נוספים. פלט JSON עם השדות שחולצו ואימות ת.ז. ישראלית. הרצה: `python scripts/extract_form_fields.py --help`

### קובצי עזר
- `references/israeli-form-types.md` - קטלוג מפורט של סוגי טפסים ממשלתיים ישראליים (טאבו/מקרקעין, טפסי רשות המסים, מסמכי ביטוח לאומי) עם תיאורי שדות, ביטויי regex לחילוץ, כללי אימות ת.ז., פורמטי תאריך/מטבע, וטיפים ל-OCR לפי פריסת הטופס. תסתכלו בו כשאתם מזהים טופס לא מוכר או בונים לוגיקת חילוץ שדות למסמך מסוים.

## מנועי OCR אפשריים

Tesseract הוא בסיס חינמי טוב לדפוס עברי, אבל ממשקי Vision מודרניים בענן משתווים אליו או עדיפים עליו בסריקות רועשות ובטפסים חתומים. תבחרו לפי השימוש:

| מנוע | דפוס עברי | כתב יד | מודל עלות | מתי להשתמש |
|------|-----------|-------|-----------|-------------|
| Tesseract (heb+eng, LSTM) | טוב לסריקות נקיות ב-300 DPI+ | חלש | חינם, ריצה מקומית | ברירת מחדל לצינורות מקומיים, לנתונים רגישים לפרטיות |
| Google Cloud Vision – Document Text Detection | טוב מאוד, עמיד לרעש | חלקי (כתב יד דמוי דפוס בלבד) | תשלום לפי בקשה | סריקות באיכות מעורבת, אצווה גדולה, טפסי PDF |
| AWS Textract (AnalyzeDocument) | טוב; חזק יותר בטפסים וטבלאות | חלקי | תשלום לפי עמוד | טפסים עם שדות מובנים, חילוץ מפתח-ערך |
| Azure AI Vision – Read API | טוב מאוד, מודע לפריסה | חלקי | תשלום לפי טרנזקציה | סביבות Azure ארגוניות, PDF חתום |
| Claude Vision (claude-sonnet-4-6 / opus-4-6) | טוב מאוד, מודע להקשר | טוב (עם הנחיה בפרומפט) | לפי טוקנים | פריסות טפסים חריגות, אימות צולב של שדות, כשרוצים שהמודל גם ינתח את הנתונים |

הערה: אף אחד מהמנועים אינו אמין לכתב יד קורסיבי על טפסים כמו נסחי טאבו ישנים. לאלה תסמנו לבדיקה ידנית במקום לנסות חילוץ אוטומטי.

## מלכודות נפוצות
- דיוק OCR בעברית יורד משמעותית בטקסט בכתב יד, במיוחד באותיות ו', ז' ו-י' שנראות דומות. תמיד תכללו ציוני ביטחון ותסמנו תווים עם ביטחון נמוך.
- טפסים ממשלתיים ישראליים סרוקים לפעמים מכילים טקסט מעורב עברית וערבית. סוכנים עלולים לזהות את החלקים הערביים כעברית או לדלג עליהם לגמרי. שתי השפות RTL אבל בטווחי Unicode שונים.
- מספרי תעודת זהות ומספרי טלפון שחולצו ב-OCR צריכים אימות עם אלגוריתמי ספרת ביקורת אחרי החילוץ, כי שגיאות OCR של ספרה בודדת נפוצות.
- הרבה טפסים ישראליים משתמשים בהדפסת מטריצת נקודות (dot-matrix) בעברית, שיש לה דיוק OCR נמוך יותר מטקסט מודפס בלייזר. סוכנים עלולים לדווח על ביטחון גבוה מדי במסמכים ממשלתיים ישנים.

## פתרון בעיות

### שגיאה: "חבילת שפה עברית של Tesseract לא נמצאה"
סיבה: קובץ traineddata בעברית לא מותקן
פתרון: תתקינו עם `sudo apt-get install tesseract-ocr-heb` (Ubuntu) או `brew install tesseract-lang` (macOS). אימות עם `tesseract --list-langs`.

### שגיאה: "פלט ה-OCR משובש או הפוך"
סיבה: בעיה בסדר של טקסט דו-כיווני
פתרון: תשתמשו בפונקציה `normalize_bidi_text()`. תוודאו ש-Tesseract משתמש ב-`--oem 1` (מצב LSTM). תבדקו שהתמונה לא מראתית.

### שגיאה: "דיוק נמוך בחלקים מסוימים של הטופס"
סיבה: איכות סריקה ירודה, חותמות/חתימות חופפות לטקסט, רקעים צבעוניים
פתרון: תגבירו את העיבוד המקדים - הסרת רעש חזקה יותר, חיתוך לאזורים ספציפיים בטופס, חילוץ מבוסס ROI לפריסות טפסים מוכרות.
